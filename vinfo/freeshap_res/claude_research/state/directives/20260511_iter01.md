# 지시 (iter 01) — Label-aware eigenvector selection 탐색

## 연구 동기 (사용자 직관)

LRFShap (`references/lrfshap.pdf`) 의 Algorithm 1 은 K(X_N, X_N) 의 best
rank-r approximation, 즉 **top-r by eigenvalue λ_i** 를 사용. 그런데 §5.5
(p.7-8) 가 보여주듯 같은 rank r 에서 dataset 별 LC(r) 와 LC(r)/FC(r) 가
천차만별이고, 이게 ER 의 cross-dataset 변동을 잘 설명함.

**핵심 질문**: LC 또는 LC/FC 가 낮은 게 ER 이 낮은 원인이라면, 같은 budget r
에서 LC 또는 LC/FC 를 최대화하도록 r 개의 eigenvector 를 선택하면 ER 도 함께
올라가는가? 이런 류의 연구가 prior work 에 존재하는가? 그리고 LC 또는 LC/FC 를
최대화하는 알고리즘은 어떻게 설계할 수 있는가?

## 이론적 sneak peek

§5.5 의 eq. (9) (p.7) 이 직접 motivate:

  ‖f^FNTK_ρ(X_N) − f^LRNTK_ρ(X_N)‖² = Σ_{i>r} (λ_i/(λ_i+ρ))² · (u_i^⊤ Y_N)²

이 bound 의 minimizer 는 top-r by λ_i 가 아닌, 점수
**s_i := (λ_i/(λ_i+ρ))² · (u_i^⊤ Y_N)²** 의 top-r. 즉 LRFShap 의 자체 bound 가
supervised eigenvector selection 을 직접 정당화.

## Agent 역할 (이번 iter 한정)

executor **사용 안 함** (실험 없음). planner ↔ critic 만 2 round.

### planner (round 1, 2)

**역할**: 알고리즘 후속 + 이론 분석 + 선행 연구 mapping 제안.

**의무**:

1. eq. (9) 가 motivate 하는 supervised score s_i = (λ_i/(λ_i+ρ))² · (u_i^⊤ Y)²
   를 식 단위로 도출. top-r by λ vs supervised top-r 의 차이를 한 단락으로.
2. **알고리즘 후보 정확히 3 개** 제시. 단순 → 정교 순서. 각 후보:
   - 정의 (input → procedure → output)
   - computational cost
   - 이론적 근거 (eq. (9) 와의 관계)
   - 위험 / 한계
3. 선행 연구 — 다음 **4 트랙에서 트랙당 1-2 편**만 인용 (총 5-7 편):
   - **Track A (kernel-side, label 무관)**: ridge leverage scores —
     Alaoui-Mahoney NeurIPS 2015 또는 Bach JMLR 2013.
   - **Track B (kernel-target alignment)**: Cristianini et al. NeurIPS 2001
     원조, Cortes-Mohri-Rostamizadeh JMLR 2012 후속 중 1편.
   - **Track C (eigenlearning / spectrum-dependent generalization)**:
     Canatar-Bordelon-Pehlevan Nature Comm 2021 또는 Simon et al. Nature Comm
     2023 중 1편.
   - **Track D (data Shapley + low-rank)**: FreeShap (이미 baseline),
     선택적으로 TRAK ICML 2023 또는 DataInf ICLR 2024 1편.
4. 각 후보 알고리즘이 4 트랙과 어떻게 다른가 한 줄씩 mapping.
5. multi-class y 처리 + partial decomp (top-2r 만 계산 후 r 재선택) 같은 실용
   issue 한 단락.

**분량**: 한국어 plan.md ≈ 1500-2500 단어 (markdown). 길게 늘이지 말 것.

**output**: round 1 → `state/iteration_01/plan.md` / round 2 → `plan_v2.md`.

### critic (round 1, 2)

**역할**: planner 산출물 검증.

**의무**:

1. 알고리즘 3 후보의 **novelty** — 이미 기존 변형 (예: Track A 의 leverage
   score) 과 동등한 항목이 섞여있지 않은지.
2. **인용 정확성** — 논문 제목 / 저자 / 연도 / venue / 주장 매핑.
3. **숨겨진 가정 / trade-off** — eq. (9) 의 S=N 한정, in-sample 한정, ρ>0 가정,
   multi-class 정의 well-definedness, partial decomp 의 cost 추정 정확성.
4. **반례 / falsifier 1-2 개** — 어떤 spectrum 또는 label 구조에서 supervised
   selection 이 오히려 ER 을 낮출 가능성?

**분량**: critique.md ≈ 800-1500 단어. critique_v2 는 final sign-off 이라 ≈
500 단어.

**output**: round 1 → `state/iteration_01/critique.md` / round 2 →
`critique_v2.md`.
