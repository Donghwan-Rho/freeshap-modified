# 2026-05-15 — iter 04 의 사용자 지시 (controlled imbalance 실험 우선)

iter 03 의 §6 에 우선 task 로 둔 controlled imbalance 실험 (Cond 2 mechanism
의 isolated 검증) 을 iter 04 본격 시작 전에 먼저 돌린다. iter 03 framework
의 가장 큰 evidence 약점 (MRPC 1 example) 을 해결하는 게 목적.

## 지시 내용

- 데이터셋: **SST-2** (node04), **MR** (node03) — 둘 다 현재 top-r 우세
  (iter 02 wins/42 = 8, 4) 이고 train_majority ≈ 0.5 (SST-2 0.562, MR 0.502)
  로 imbalance 가 깊지 않음.
- Train class ratio: **50/50 (기존 결과 재사용), 70/30 (신규), 90/10 (신규)**.
  positive class fraction = ratio 의 첫 숫자. n_train = 2000 고정.
- Methods: **LRFShap** (top-r by λ) + **A1** (top-r by supervised score).
- Approximate: **inv** (rank-independent baseline) + **eigen** rank ∈ {1, 5,
  10, 15, 20, 25, 30}%.
- Seed: 2026 (iter 02 와 동일 single seed).
- TMC iter 500, ρ_eigen = 1e-2, λ_inv = 1e-6.

## 성공 기준

- 70/30 / 90/10 에서 LR_predict_majority_frac 이 train_majority 보다 더
  majority 편향 되는지 (= Cond 2 의 *effect-side* trigger).
- 동일 (r, sel%) 셀에서 LR 대비 A1 격차가 imbalance 깊이와 함께 증가하는지.
  50/50 의 top-r 우세 (8/42, 4/42) 가 70/30 또는 90/10 에서 *어느 시점에서*
  뒤집히는지. 뒤집힘 → Cond 2 mechanism 의 directly causal 검증 성공.
- 안 뒤집히면 → Cond 2 가설 *불완전*, PR(c²) 등 다른 hidden variable 추가
  필요.
